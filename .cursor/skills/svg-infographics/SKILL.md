# SVG Infographics Skill

You produce **stunning, publication-quality SVG infographics** — the kind that get shared, screenshotted, and saved. Every output should feel like it was designed by a skilled information designer, not generated by an AI.

---

## Phase 1: Design Thinking (Do This Before Writing Any Code)

Ask yourself — or the user — these questions:

1. **What story does this data tell?** Find the narrative, not just the facts.
2. **What's the emotional tone?** Authoritative/corporate? Playful/educational? Bold/editorial? Elegant/luxury?
3. **What type of infographic fits best?** (See types below)
4. **Who's the audience?** Determines density, terminology, color choices.
5. **What's the ONE thing they must remember?** Make that the visual anchor.

**Commit to a bold aesthetic direction before writing a single line of SVG.**

---

## Infographic Types

| Type | When to Use |
|------|-------------|
| **Statistical / Data** | Numbers, percentages, rankings, comparisons |
| **Timeline** | Historical events, milestones, process steps over time |
| **Process / Flow** | How something works, step-by-step, cause-effect |
| **Comparison** | Side-by-side, versus, pros/cons |
| **Geographic** | Location-based data (use simplified SVG maps) |
| **Anatomy / Explainer** | Labeled diagrams, how something is structured |
| **List / Hierarchy** | Top N items, rankings, organizational structure |
| **Narrative** | Story-driven, mixed text + visuals |

## SVG Technical Rules

### Canvas Setup
```svg
<svg viewBox="0 0 800 1200" xmlns="http://www.w3.org/2000/svg" 
     font-family="system-ui, sans-serif">
  <defs><!-- gradients, filters, patterns here --></defs>
  <!-- content -->
</svg>
```

**Standard widths:**
- `800` — standard infographic (portrait)
- `1200` — wide / landscape
- `600` — narrow / social media square-ish

**Always use `viewBox`** — never fixed `width`/`height` only, so it scales.

### Typography System

Pick 1–2 font stacks and stay consistent. Use `<style>` for reuse:

```svg
<style>
  .title { font-size: 36px; font-weight: 800; letter-spacing: -1px; }
  .subtitle { font-size: 16px; font-weight: 400; opacity: 0.7; }
  .label { font-size: 13px; font-weight: 600; text-transform: uppercase; letter-spacing: 1px; }
  .body { font-size: 14px; font-weight: 400; line-height: 1.5; }
  .number { font-size: 48px; font-weight: 900; }
  .caption { font-size: 11px; opacity: 0.5; }
</style>
```

**Font stacks by mood:**
- **Geometric/Modern**: `'DM Sans', 'Inter', system-ui, sans-serif`
- **Editorial/Magazine**: `Georgia, 'Times New Roman', serif`
- **Technical/Data**: `'IBM Plex Mono', 'Courier New', monospace`
- **Friendly/Accessible**: `'Nunito', 'Trebuchet MS', sans-serif`

Google Fonts can be embedded via `<defs>`:
```svg
<defs>
  <style>@import url('https://fonts.googleapis.com/css2?family=DM+Sans:wght@400;700;900');</style>
</defs>
```

### Color Systems

Always define a palette in `<defs>`. Never use raw hex values scattered through the SVG.

**Dark editorial palette:**
```
Background: #0F0F14   Surface: #1A1A24   Accent: #FF6B35
Text: #FFFFFF         Muted: #888899     Success: #4ECDC4
```

**Light professional palette:**
```
Background: #FAFAFA   Surface: #FFFFFF   Accent: #2563EB
Text: #111827         Muted: #6B7280     Highlight: #FEF3C7
```

**Bold data palette (categorical):**
```
#FF6B6B  #4ECDC4  #45B7D1  #96CEB4  #FFEAA7  #DDA0DD
```

Use `<linearGradient>` for bars, backgrounds, and accent elements.

### Layout Grid

Use a mental 12-column grid. Common layouts:

```
Header (full width)
─────────────────────────────────────────
[  Key Stat  ] [  Key Stat  ] [  Key Stat ]   ← 3-col row
─────────────────────────────────────────
[     Main Chart / Visual (full)        ]
─────────────────────────────────────────
[ Left Col 60%      ] [ Right Col 40%  ]
─────────────────────────────────────────
Footer / Source
```

Use `x`, `y`, `width`, `height` math consistently. Define variables as comments:
```svg
<!-- LAYOUT: margin=40, col=240, gutter=20 -->
```

## Visual Excellence Rules

### 1. Hero Numbers
Large stats should be HUGE and bold — make them unmissable:
```svg
<text x="400" y="200" text-anchor="middle" class="number" fill="#FF6B35">87%</text>
<text x="400" y="228" text-anchor="middle" class="label" fill="#888899">OF USERS AGREE</text>
```

### 2. Charts — Do Them Right

**Bar Charts:**
- Round bar corners with `rx="4"`
- Add value labels above bars
- Gridlines at 25% intervals, `opacity="0.15"`
- Animate with `<animate>` if the SVG will be viewed in browser

**Donut/Pie Charts:**
- Use `stroke-dasharray` technique (more reliable than `<path>`)
- Always show the legend with colored squares
- Add center label for the primary value

**Progress Bars:**
- Background track, colored fill, end cap rounding
- Percentage label right-aligned

**Timelines:**
- Vertical line with nodes
- Alternate left/right for content (zigzag) for density
- Use icons or numbers in circles as node markers

### 3. Backgrounds & Depth

Don't use flat white. Options:
```svg
<!-- Subtle grid -->
<pattern id="grid" width="20" height="20" patternUnits="userSpaceOnUse">
  <path d="M 20 0 L 0 0 0 20" fill="none" stroke="#333" stroke-width="0.5" opacity="0.3"/>
</pattern>

<!-- Noise/grain filter -->
<filter id="grain">
  <feTurbulence type="fractalNoise" baseFrequency="0.65" numOctaves="3" stitchTiles="stitch"/>
  <feColorMatrix type="saturate" values="0"/>
  <feBlend in="SourceGraphic" mode="overlay" result="blend"/>
</filter>

<!-- Radial glow -->
<radialGradient id="glow" cx="50%" cy="30%" r="60%">
  <stop offset="0%" stop-color="#2563EB" stop-opacity="0.15"/>
  <stop offset="100%" stop-color="#0F0F14" stop-opacity="0"/>
</radialGradient>
```

### 4. Decorative Accents

Small details that elevate the design:
- Thin horizontal rule `<line>` with gradient fade
- Dot grid patterns in corner areas
- Colored left-border on callout boxes (`<rect>` with accent color, narrow width)
- Small geometric shapes (circles, triangles) as visual interest
- Icon-like shapes using `<path>` for section markers

### 5. Callout / Highlight Boxes

```svg
<!-- Accent left-border box -->
<rect x="40" y="300" width="4" height="80" fill="#FF6B35" rx="2"/>
<rect x="52" y="300" width="708" height="80" fill="#1A1A24" rx="8"/>
<text x="72" y="334" class="label" fill="#FF6B35">KEY INSIGHT</text>
<text x="72" y="356" class="body" fill="#CCCCDD">The main message goes here in readable body text.</text>
```

### 6. Icons via SVG Paths

Use simple geometric icon shapes:
```svg
<!-- Checkmark circle -->
<circle cx="20" cy="20" r="16" fill="#4ECDC4" opacity="0.2"/>
<circle cx="20" cy="20" r="16" fill="none" stroke="#4ECDC4" stroke-width="1.5"/>
<path d="M13 20 L18 25 L27 15" stroke="#4ECDC4" stroke-width="2" fill="none" stroke-linecap="round"/>

<!-- Arrow right -->
<path d="M10 20 L26 20 M20 14 L26 20 L20 26" stroke="currentColor" stroke-width="2" fill="none" stroke-linecap="round" stroke-linejoin="round"/>
```

## Header / Footer Conventions

**Header:**
```svg
<!-- Background bar -->
<rect x="0" y="0" width="800" height="100" fill="url(#headerGrad)"/>
<!-- Eyebrow label -->
<text x="40" y="38" class="label" fill="#FF6B35" opacity="0.9">INFOGRAPHIC</text>
<!-- Title -->
<text x="40" y="68" class="title" fill="white">Main Title Here</text>
<!-- Optional subtitle on right -->
<text x="760" y="68" class="subtitle" fill="white" text-anchor="end">Subtitle or date</text>
```

**Footer / Source line:**
```svg
<line x1="40" y1="1170" x2="760" y2="1170" stroke="#444" stroke-width="0.5"/>
<text x="40" y="1188" class="caption" fill="#666">SOURCE: Data source name, Year</text>
<text x="760" y="1188" class="caption" fill="#666" text-anchor="end">© Author / Organization</text>
```

## Quality Checklist

Before outputting, verify:

- [ ] **Readable at thumbnail size** — title and key stat visible when small
- [ ] **Color contrast** — text always ≥ 4.5:1 against background  
- [ ] **Consistent spacing** — use multiples of a base unit (8px or 10px)
- [ ] **No orphaned numbers** — every data point has a label
- [ ] **Source attribution** — footer cites data origin
- [ ] **Intentional hierarchy** — 3 levels max: headline → section → detail
- [ ] **One clear focal point** — the eye knows where to land first
- [ ] **No raw `#hex` colors** in the SVG — use named fills or `<defs>`

## Output Format

Deliver the SVG as a file via `present_files`, or as an inline artifact if in Cursor.ai. 

Always include:
1. The complete SVG (no placeholders or `<!-- TODO -->` comments)  
2. A brief note on the color palette and design choices made
3. An offer to adjust colors, layout, or data

The SVG must be **self-contained** — no external dependencies except optional Google Fonts.
